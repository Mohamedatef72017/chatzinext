Context

A new tenant/account was created.

The tenant was given its own Knowledge Base data.

However, the AI bot does NOT appear to use the Knowledge Base.

Instead it always replies like:

“أنا شاتزي، المساعد الافتراضي لشركة مواتف. كيف يمكنني مساعدتك اليوم؟”

Problems observed:

1. The bot seems to use tenant/account name instead of Knowledge Base content.
2. It transliterates or Arabizes names incorrectly:
    * bot name should remain: chatzi
    * account name should remain: moatef
    * It should NOT translate or Arabize brand/account names unless explicitly configured.
3. The AI frequently pauses/stops itself.
4. The bot must NEVER stop itself except in exactly two cases:
    * a human agent sends a message in the conversation
    * the user manually disables/pauses the bot using the UI button if such button exists
5. AI replies are very slow.
6. The AI docs and the actual AI implementation may be out of sync after the recent billing and architecture refactors.
7. Billing upgrade fails with:
    Invalid API Key provided: sk_test_*******************************************************************************************************mhKW
8. This is very suspicious because billing should use Stripe keys, not AI provider keys.
9. The system may have hidden conflicts introduced by recent refactors.

* aiStatus

* human
* escalat
* takeover

Rules:

The bot may pause ONLY when:

1. A human agent sends a message in the conversation.
2. A human user/admin explicitly pauses/disables the bot from the UI/API.

The bot must NOT pause itself because:

* it did not know the answer
* it created a ticket
* it escalated
* it detected sentiment
* it failed to retrieve knowledge
* it reached a fallback
* it misunderstood intent
* it asked for human help
* the customer asked a difficult question

If the customer asks for a human:

* the AI may create a ticket or mark handoff requested
* but the bot should not permanently pause unless a human actually replies or the manual pause endpoint is used.

Fix all incorrect pause/handoff behavior.

Preserve valid human takeover behavior.

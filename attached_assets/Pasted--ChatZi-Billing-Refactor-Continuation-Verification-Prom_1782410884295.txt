# What Was Supposed To Be Done

The previous task was to refactor ChatZi billing from a hardcoded AI-message-only system into an enterprise-grade, feature-based, configuration-driven billing architecture.

The target architecture:

1. BillingPlan must become the single source of truth.
2. Plan features must be stored in `BillingPlan.features[]`.
3. Runtime billing and entitlements must NOT depend on hardcoded `PLAN_DEFAULTS`.
4. `PLAN_DEFAULTS` may remain only as a migration fallback or seed helper, not as the real runtime decision source.
5. Usage metering must become generic.
6. AI message quota must work through the generic usage engine.
7. Future billable features must be addable without rewriting core billing logic.
8. Super Admin must be able to manage plans, features, limits, and billing controls.
9. Stripe must keep working.
10. Payment provider abstraction must exist for future providers.
11. Wallet/Credits foundation must exist safely.
12. Documentation must be updated in docs.

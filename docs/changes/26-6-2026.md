# ChatZi Engineering Change Log — 26 June 2026

## Summary

تحسينان متكاملان مُصمَّمان خصيصاً لتعزيز أداء **GPT-4.5 nano** (النموذج الخفيف المستخدم في الإنتاج):

## التغيير الأول — Response Formatting

### الملف: `src/lib/ai/build-system-prompt.ts`

أُضيف قسم **RESPONSE FORMATTING** جديد في `GLOBAL_CRM_SYSTEM_PROMPT` مباشرةً بعد قسم RESPONSE QUALITY.

**القواعد الخمس المُضافة:**

**SEPARATE YOUR IDEAS:**
كل فكرة مستقلة في فقرتها الخاصة مع سطر فارغ بين الفقرات. ممنوع دمج أفكار متعددة في كتلة نص واحدة.

**USE LISTS WHEN APPROPRIATE:**
- 3 عناصر أو أكثر (خدمات، أسعار، خطوات، خيارات) → قائمة bullet point، عنصر واحد لكل سطر
- تسلسل خطوات يتبعها العميل → قائمة مُرقَّمة (1. → 2. → 3.)
- ممنوع ضغط القائمة في جملة واحدة بفواصل

**KEEP PARAGRAPHS SHORT:**
2–3 جمل لكل فقرة كحدٍّ أقصى. إذا احتاج الموضوع تفصيلاً، يُقسَّم إلى فقرات قصيرة متعددة.

**NEVER SEND A WALL OF TEXT:**
إذا تجاوز الرد 4 أسطر بدون سطر فارغ → كثيف جداً، اقسمه.

**DO NOT USE MARKDOWN DOCUMENT SYNTAX:**
ممنوع ## أو ### headers أو horizontal rules. هذه واجهة محادثة وليست مستنداً.

**المكان في الملف:** بعد سطر `Do not write the same closing line in every message. Vary your language naturally.` وقبل قسم KNOWLEDGE COMPLIANCE.

## التغيير الثاني — Intent Signal Injection

### الملف: `src/mastra/workflows/ai-reply.workflow.ts`

#### تحديث الـ Import

```typescript
// قبل
import { detectBusinessIntent, isDirectKnowledgeIntent } from "@/lib/ai/business-intent";

// بعد
import {
  detectBusinessIntent,
  isDirectKnowledgeIntent,
  isPurchaseReadyIntent,
  isHighSensitivityIntent,
  type BusinessIntent
} from "@/lib/ai/business-intent";
```

#### تحديث `buildRuntimeContext`

إضافة حقن الـ signal مباشرةً بعد `businessIntent`:

```typescript
if (inputData.businessIntent) {
  parts.push(`businessIntent=${inputData.businessIntent}`);

  const intent = inputData.businessIntent as BusinessIntent;

  if (isPurchaseReadyIntent(intent)) {
    parts.push(
      "intentSignal=purchase_ready → customer is close to booking or buying; " +
      "skip re-explaining services; move directly to the next concrete action; " +
      "close warmly and concisely"
    );
  } else if (isHighSensitivityIntent(intent)) {
    parts.push(
      "intentSignal=high_sensitivity → customer needs careful emotional handling; " +
      "lead with empathy and acknowledgment before anything else; " +
      "do NOT upsell or cross-sell; prioritize resolution over all else"
    );
  }
}
```

#### كيف يصل الـ signal للنموذج

```
[knowledgeStep]
      ↓
detectBusinessIntent(message) → "appointment" | "complaint" | ...
      ↓
[generateReplyStep]
      ↓
buildRuntimeContext(inputData)
      ↓
contextSummary = "businessIntent=appointment; intentSignal=purchase_ready → ..."
      ↓
buildUnifiedSystemPrompt({ contextSummary })
      ↓
Runtime context section in system prompt — يراه GPT-4.1 nano
```

#### التأثير المباشر على ردود GPT-4.5 nano

| الرسالة | intent | intentSignal | السلوك المتوقع |
|---|---|---|---|
| "بكام الخدمة؟" | `prices` | `purchase_ready` | يُجيب بالسعر + خطوة الحجز مباشرة |
| "عايز أحجز موعد" | `appointment` | `purchase_ready` | يبدأ خطوات الحجز فوراً |
| "عندي مشكلة" | `support` | `high_sensitivity` | يبدأ بالإقرار العاطفي، لا upsell |
| "عندي شكوى" | `complaint` | `high_sensitivity` | تعاطف أولاً، حل ثانياً، لا بيع |
| "إيه الخدمات؟" | `services` | *(لا signal)* | سلوك عادي |

## Verification Steps

### Response Formatting
1. اطلب من البوت سرد الخدمات → يجب أن يكون الرد قائمة bullet، وليس جملة واحدة بفواصل.
2. اطلب خطوات الحجز → يجب أن تكون الإجابة قائمة مُرقَّمة 1. 2. 3.
3. رسالة سؤال عام به نقطتان منفصلتان → تحقق من وجود سطر فارغ بينهما.
4. `grep -n "RESPONSE FORMATTING" src/lib/ai/build-system-prompt.ts` → يجب أن يُعيد نتيجة.

### Intent Signal Injection
1. أرسل رسالة "عايز أحجز موعد" → تحقق من أن context الـ agent يحتوي `intentSignal=purchase_ready`.
2. أرسل رسالة "عندي شكوى" → تحقق من أن context يحتوي `intentSignal=high_sensitivity`.
3. أرسل رسالة "إيه الخدمات؟" → تحقق من عدم وجود `intentSignal` في context.
4. `isPurchaseReadyIntent("appointment")` → `true`. `isPurchaseReadyIntent("services")` → `false`.
5. `isHighSensitivityIntent("complaint")` → `true`. `isHighSensitivityIntent("prices")` → `false`.

## Risks / Notes

1. **intentSignal لا يُعاد حقنه في `unifiedPrompt` المُخزَّن في cache** — الـ `intentSignal` يُحقن في `buildRuntimeContext` وقت الـ generate step مباشرة وليس في الـ cached prompt. هذا صحيح: الـ signal ديناميكي لكل رسالة بينما الـ prompt الأساسي ثابت ومُخزَّن. لا تحتاج لإبطال الـ cache عند تغيير intent.
2. **RESPONSE FORMATTING + GPT-4.5 nano** — قواعد التنسيق فعّالة بشكل خاص للنماذج الصغيرة. للنماذج الأكبر مثل GPT-4o، نفس القواعد تعمل مع هامش خطأ أصغر. لا أثر سلبي عند التبديل لنموذج أكبر مستقبلاً.
3. **زيادة حجم `GLOBAL_CRM_SYSTEM_PROMPT`** — إضافة RESPONSE FORMATTING رفعت الـ prompt لـ ~200 سطر. Prompt cache يُخفف الأثر: الـ prompt يُبنى مرة واحدة كل 5 دقائق لكل `(bot, settings_version)`.
